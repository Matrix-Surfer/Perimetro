## La señal

Microsoft confirmó que el programa de actualizaciones de seguridad extendidas (ESU) para Exchange Server 2016 y 2019 concluye en octubre de 2026, sin posibilidad de una nueva prórroga; ese plazo ya había sido extendido una vez, en abril de 2026. A partir de octubre, cualquier organización que siga usando esas versiones en sus propios servidores dejará de recibir correcciones de seguridad, sin importar qué tan grave sea una falla que se descubra después.

## El supuesto que se rompe

La suposición de que el software de correo corporativo seguirá recibiendo parches de seguridad mientras la organización lo siga usando activamente. El soporte tiene una fecha de vencimiento independiente de si el sistema sigue en uso o no, y esa fecha ya está fijada.

## Qué observar

Si la organización usa Exchange Server 2016 o 2019 instalado en sus propios servidores, y si ya existe un plan de migración con fecha concreta antes de octubre de 2026, ya sea hacia una versión más reciente o hacia un servicio de correo en la nube. Migrar un sistema de correo completo toma meses de planeación; la fecha límite ya no es una posibilidad lejana.
